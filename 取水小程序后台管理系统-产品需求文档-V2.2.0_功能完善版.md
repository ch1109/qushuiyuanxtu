## 一、产品概述

### 1.1 产品定位与重构背景

**产品定位**：取水小程序后台管理系统是一套面向平台运营人员和渠道合伙人的管理工具，用于管理自助取水设备、用户、订单、合伙人及财务等核心业务。系统追求**简洁高效**，支持多角色、多权限、多区域的精细化管理。

**重构背景**：
- **现有系统现状**：当前后台基于ThinkPHP 5.x + FastAdmin框架，包含14大功能模块、80+个控制器、11个插件，功能繁多但操作复杂
- **重构目标**：在保留核心业务逻辑和数据库的基础上，重新设计用户界面和交互流程，将功能模块从14个精简至6个，页面从80+精简至17个核心页面
- **重构策略**：采用"优化现有系统"的方式，共用重构后的数据库（56张表），淘汰低频功能，优化高频操作，提升移动端体验
- **与旧系统关系**：新系统上线后，旧系统保留1个月作为备份，期间新旧系统并行运行，确保平稳过渡

### 1.2 目标用户

| 角色 | 职责 | 核心关注点 | 权限范围 |
|-----|------|-----------|---------|
| 超级管理员 | 平台全局管理 | 整体运营数据、系统配置 | 全部功能 |
| 运营人员 | 日常业务处理 | 用户咨询、订单处理、内容配置 | 业务操作 |
| 财务人员 | 资金管理 | 提现审批、流水明细、收入报表 | 财务相关 |
| 设备运维 | 设备维护 | 设备状态、故障处理 | 设备相关 |
| 渠道合伙人 | 管理所属区域 | 自己区域的设备和收益数据 | 限定区域 |

### 1.25 核心业务逻辑（重要） 🆕

**水币结算机制**：

```
┌─────────────────────────────────────────────────────────────────┐
│                      取水业务核心逻辑                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1️⃣ 水币购买                                                     │
│     • 标准售价：0.05元/水币（普通用户）                          │
│     • VIP售价：0.025元/水币（超级会员5折）                       │
│     • VIP年费：198元/年，开通赠送2000水币                        │
│     • 支付方式：微信/支付宝                                      │
│                                                                 │
│  2️⃣ 取水消费                                                     │
│     • 用户取水只能使用水币支付                                   │
│     • 不支持微信/支付宝直接支付取水                              │
│     • 1水币 = 1升水（固定比例）                                  │
│     • 水币按整数扣费（如取2.5升，扣3水币）                        │
│     • 门关闭后统一结算，非实时扣费                               │
│                                                                 │
│  3️⃣ 三账户体系                                                   │
│     • 充值币：用户充值的水币，可申请退款                          │
│     • 赠送币：注册/活动赠送，不可退款                            │
│     • 奖励币：推广奖励，不可退款                                 │
│     • 扣费优先级：奖励币 → 赠送币 → 充值币                       │
│                                                                 │
│  4️⃣ 分润结算                                                     │
│     • 按水币数量计算分润，非取水量                               │
│     • 每台设备可绑定多个合伙人（7类各1个）                       │
│     • 每个合伙人可绑定多台设备（多对多）                         │
│     • 每月1日自动结算到合伙人佣金账户                            │
│                                                                 │
│  5️⃣ 设备特性                                                     │
│     • 设备接自来水管网，无水位概念                               │
│     • 无需补水，无低水位预警                                     │
│     • 状态：在线/离线/故障/维护                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**关键规则**：
- ✅ 水币是唯一的取水支付方式
- ✅ 水币按整数取整扣费
- ✅ 门关闭后统一结算
- ✅ 合伙人与设备多对多关系
- ❌ 不支持微信/支付宝直接取水
- ❌ 设备无水位概念

| 异常预警卡片 | 离线设备、故障设备、待处理事项 | 点击跳转对应页面 | 今日实时(设备状态表) |

- **今日实时数据**：从Redis缓存读取或直接聚合订单表，每5分钟自动刷新，限制手动刷新频次(最多1分钟1次)
- **历史统计数据**：从wdy_stats_daily表读取，每日凌晨1点自动汇总前一天数据
- **排行榜数据**：基于历史daily表统计，每小时更新一次缓存

**数据库关联**：
- 今日实时：`wdy_order`(当日订单) + `wdy_device`(设备状态) + Redis缓存
- 历史数据：`wdy_stats_daily`(按日汇总的统计表)

**运维管理标签页详细说明**：

**1. 维护记录**
- 数据来源：`wdy_device_maintenance` 表
- 显示内容：维护类型（维修/更换/清洁/巡检）、维护说明、更换零件、维护成本、维护人员
- 操作功能：添加维护记录、查看附件照片、下次维护提醒
- 支持字段：maintenance_type, description, parts_replaced, cost, operator_name, attachment_urls

**2. 故障工单** 🆕
- 数据来源：`wdy_device_fault` 表
- 显示内容：故障类型、故障描述、优先级、处理状态、处理人、处理结果
- 操作功能：创建工单、分配维修人员、上传故障图片、记录处理结果
- 支持字段：fault_type, fault_desc, images, priority, status, handler_id, handle_result

**3. 调试任务** 🆕
- 数据来源：`wdy_device_debug` 表
- 显示内容：调试类型、调试参数、任务状态、调试结果、调试日志
- 操作功能：创建调试任务、查看调试日志、下载调试结果
- 支持字段：debug_type, debug_params, status, result, logs

**操作日志标签页扩展说明**：

**1. 远程指令记录** 🆕
- 数据来源：`wdy_device_cmd` 表
- 显示内容：指令类型（重启/参数设置/固件升级）、指令参数、执行状态、执行结果
- 筛选功能：按指令类型、执行状态、时间范围筛选
- 支持字段：cmd_type, cmd_params, status, result, operator_id

**2. 状态变更记录** 🆕
- 数据来源：`wdy_device_status_log` 表
- 显示内容：原状态、新状态、变更原因、操作人
- 追溯功能：完整的设备状态变更历史链
- 支持字段：old_status, new_status, change_reason, operator_id

**基本信息标签页补充**：
- 最后上报时间：来自 `wdy_device.last_report_time`
- 在线时长统计：来自 `wdy_device.online_duration`
- 设备配置JSON：查看 `wdy_device.device_config` 完整配置

**功能清单**：

| 功能 | 说明 | 使用场景 |
|-----|------|---------|
| 添加设备 | 录入设备信息、经纬度，自动生成二维码 | 新装设备时 |
| 设备二维码 | 查看/下载/打印设备专属二维码 | 用户扫码取水 |
| 远程重启 | 远程发送重启指令 | 设备故障时 |
| 参数设置 | 远程修改设备运行参数 | 调整配置时 |
| 批量操作 | 批量设为维护/恢复运行 | 批量管理时 |

**注**：以上分润比例为系统默认值，实际绑定时可根据具体情况调整，但总和不超过100%。

**分润计算逻辑**：

```
示例：用户消费100水币（门关闭后结算）

假设绑定了3个合伙人：
- 地址合伙人 20% → 100 × 20% = 20水币 → 20×0.05元 = 1.00元
- 出资合伙人 30% → 100 × 30% = 30水币 → 30×0.05元 = 1.50元  
- 技术合伙人 10% → 100 × 10% = 10水币 → 10×0.05元 = 0.50元

平台收益：100水币 - 20 - 30 - 10 = 40水币 → 40×0.05元 = 2.00元 (40%)

用户消费金额：100水币（可能来自充值币、赠送币或奖励币的任意组合）
合伙人佣金总额：3.00元
平台净收入：2.00元

注意：
1. 结算单位是水币，不是升
2. 用户消费完成（门关闭）后才统一扣费
3. 分润按水币数量计算，然后转换为金额
4. 每月1日自动结算到合伙人账户
```

**合伙人绑定规则**（重要）：

- ⚠️ 同一设备的同一类型只能绑定1个合伙人
- ⚠️ 所有分润比例总和不超过100%

```
6. 更换后水质恢复正常，记录处理结果

场景4：用户投诉处理
1. 用户反馈水质异常或口感变化
2. 运维人员查询该设备水质历史数据
3. 查看水质检测报告和趋势图
4. 导出最近7天的水质报告
5. 发送给用户查看，消除疑虑

场景5：监管部门检查
1. 卫生监督部门要求提供水质报告
2. 在水质检测报告列表筛选时间范围
3. 批量导出月度水质检测报告
4. 生成PDF格式提交给监管部门
5. 展示水质合格率和关键指标统计
```

**数据库关联**：
- `wdy_device_quality` - 水质检测历史数据表(存储所有水质检测记录)

  - 支持按时间范围查询趋势数据
- **水质检测报告**：基于`wdy_device_quality`数据动态生成,不单独存表
  - 报告通过PDF生成逻辑实时生成
  - 可筛选时间范围导出历史报告

  - 故障类型`fault_type`设为`WATER_QUALITY`

- `wdy_device` - 设备信息表(config字段存储滤芯配置JSON)
- `wdy_device_maintenance` - 设备维护记录表(记录滤芯更换历史)

**滤芯数据存储说明**：

  ```json
      "activated_carbon": {"install_date": "2025-10-01", "lifespan_days": 180},
      "ro_membrane": {"install_date": "2023-12-15", "lifespan_days": 720}
    }
  }
  ```
- **滤芯更换历史**：记录在`wdy_device_maintenance`表的维护记录中
  - `maintenance_type` = 'FILTER_REPLACEMENT'(滤芯更换)
  - `description`字段记录更换的滤芯类型和规格

**数据采集方式**：
- 设备内置水质传感器实时采集数据
- 每5分钟上传一次数据到服务器
- 系统自动分析数据并生成趋势
- 异常情况立即触发预警推送

**权限控制**：

- 普通用户：在小程序查看正在使用设备的水质

```
**审批流程** (金额>100元时触发)：
1. 操作人提交调整申请
2. 系统生成审批工单(wdy_wallet_adjustment_approval)
3. 通知财务人员审批
4. 财务审批通过/驳回
5. 通过后自动执行余额调整
6. 短信/站内信通知用户

**数据库关联**：
- `wdy_user` - 用户基础信息
- `wdy_user_wallet` - 三账户余额(充值币、赠送币、奖励币)
- `wdy_water_coin_log` - 水币流水记录
- `wdy_wallet_adjustment_approval` - 余额调整审批表(新增)
  - adjustment_amount: 调整金额
  - adjustment_type: 调整类型(INCREASE/DECREASE)
  - account_type: 账户类型(RECHARGE/GIVE/REWARD)
  - reason: 调整原因(TEXT)
  - status: 审批状态(PENDING/APPROVED/REJECTED)
  - applicant_id: 申请人ID
  - approver_id: 审批人ID
  - applied_at: 申请时间
  - approved_at: 审批时间
- `wdy_admin_log` - 管理员操作日志(记录完整审批流程)
- `wdy_user.bank_info_masked` - 银行卡脱敏信息
```

```
- `wdy_product_commission` - 商品佣金表
- `wdy_product_review` - 商品评价表
- `wdy_product_stock_log` - 库存流水表(新增) 🆕
  - product_id: 商品ID
  - change_type: 变动类型(IN入库/OUT出库)
  - change_quantity: 变动数量(正数入库,负数出库)
  - stock_after: 调整后库存
  - reason: 调整原因(TEXT)
  - remark: 备注说明(TEXT)
  - operator_id: 操作人ID(系统自动=-1)
  - created_at: 操作时间
```

```
│  └─────────────────────────────────────────────────────┘      │
│                                                                │
│  [刷新数据] [查看日志] [导出报告]                              │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

**数据库关联**：
- `wdy_uid_sequence` - UID序列表(P0核心表)
  - province_code: 省份代码(2位，如GD、BJ)
  - current_sequence: 当前序号(已分配到的序号)
  - max_sequence: 最大序号(默认999999)
  - version: 版本号(用于乐观锁，并发控制)
  - created_at: 创建时间
  - updated_at: 最后更新时间

**UID生成机制**：
```
1. 用户注册时需要生成唯一UID
2. 根据用户所在省份获取省份代码
3. 使用悲观锁(FOR UPDATE)锁定记录
4. 检查version是否被其他请求修改(乐观锁)
5. current_sequence + 1，version + 1
6. 组装UID = 省份代码 + 序列号(如GD123456)
7. 如果version冲突，自动重试最多3次
8. 失败则记录日志并降级处理
```

**功能特点**：
1. 高并发支持: 悲观锁+乐观锁双重保障，支持2000+ QPS
2. 自动预警: 使用率达90%时自动预警
3. 性能监控: 实时监控QPS、成功率、重试次数
4. 迁移方案: 提供序号即将用尽的迁移建议
5. 审计追踪: 完整记录每次UID生成和版本变更

### 8.1 名词解释

| 名词 | 解释 |
|-----|------|
| **三账户系统** | 用户钱包包含3个独立账户：充值币、赠送币、奖励币 |
| 充值水币 | 用户充值的水币，可申请退款，存储在recharge_coin账户 |
| 赠送水币 | 注册或活动赠送的水币，不可退款，存储在give_coin账户 |
| 奖励水币 | 推广奖励的水币，不可退款，存储在reward_coin账户 |
| 水币 | 平台虚拟货币，1水币可购买1升水，标准售价0.05元/水币，VIP售价0.025元/水币 |
| 水币结算 | 用户取水完成（门关闭）后按整数水币统一扣费，不支持微信/支付宝直接支付 |
| 扣费优先级 | 三账户扣费顺序：奖励币 → 赠送币 → 充值币 |
| VIP会员 | 超级会员，年费198元，享5折购买水币优惠，开通赠送2000水币 |
| 水币售价 | 普通用户0.05元/水币，VIP会员0.025元/水币（5折） |
| 分润 | 订单产生后，按水币数量比例分配给各级合伙人的收益 |
| 佣金账户 | 合伙人的分润收益账户，可提现，存储在money字段 |
| 促销账户 | 合伙人绑定的促销补贴账户，用于营销活动，独立核算 |
| 子账号 | 为渠道商/合伙人开设的受限后台账号 |
| 区域权限 | 基于省/市/区/设备的数据访问权限控制 |
| 区域热力图 | 用颜色深浅表示各区域消费热度的地图 |
| 用户生命周期 | 用户从注册到流失的完整阶段划分 |
| UID | 用户唯一标识，格式：省份代码(2位)+序列号(6位)，如GD000001 |
| 审批流转历史 | 充值/提现审批的完整流程记录，支持追溯 |
| 价格变更历史 | 水币售价/VIP价格的调整记录，用于审计 |
| 维护记录 | 设备保养、维修、清洁等工单记录 |

### 8.2 7类合伙人类型说明

**重要说明**：以下为系统当前配置的7种合伙人类型，具体分润比例可在后台动态调整。

| 类型 | 默认分润 | 职责说明 | 条件要求 |
|-----|---------|---------|---------|
| 营销合伙人 | 10% | 负责拍摄小视频、推广健康饮用水标准、营销卖水 | 需发起50人拼团 |
| 地址合伙人 | 20% | 提供安装地址、支持接水接电、日常保洁 | 需上传地址三张照片 |
| 出资合伙人 | 30% | 设备价值9.8万，直接出资1万元 | 成为投资合伙人 |
| 管理合伙人 | 10% | 管理审批合伙人、审批地址合理性、其他合伙人绩效考核 | 由平台指定 |
| 区域合伙人 | 10% | 拓展区域平台客户 | 一次性拓展100个网点以上 |
| 拓展合伙人 | 10% | 拓展单个网点 | 累计拓展网点50个、可升级为区域合伙人 |
| 技术合伙人 | 10% | 提供技术解答、设备耗材维护、软硬件设施维护 | 具备技术能力 |

**合伙人配置规则**：
- 同一设备可绑定多个不同类型的合伙人
- 同一类型只能绑定1个合伙人
- 所有分润比例总和不超过100%，剩余为平台收益
- 合伙人类型可在系统配置中动态调整
- 每月1日自动结算上月分润到合伙人账户

**数据库支持**：
- `wdy_partner_type_config` - 合伙人类型配置表
- `wdy_partner_profit_config` - 设备分润绑定表
- `wdy_commission_log` - 佣金流水记录表
